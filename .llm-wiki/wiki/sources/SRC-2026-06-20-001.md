# ROADMAP.md
> _Original: [https://github.com/pablo-ross/strix-halo-gmktec-evo-x2/blob/main/ROADMAP.md](https://github.com/pablo-ross/strix-halo-gmktec-evo-x2/blob/main/ROADMAP.md)_

## Summary

This document provides a comprehensive, step-by-step roadmap for configuring an Ubuntu 24.04 system to run large language models (LLMs) on the GMKTEC EVO-X2 mini PC, which features the AMD Ryzen AI Max+ 395 Strix Halo APU. The guide prioritizes optimizing ROCm 7 RC with rocWMMA and hipBLASlt for maximum inference performance, addressing the unique challenges of APU unified memory architectures. It covers BIOS tuning for GPU memory and power limits, kernel and GRUB modifications for >15GB VRAM access, and precise udev rule configurations to prevent ROCm resource errors.

The setup leverages a containerized environment using Podman and Distrobox to isolate the ROCm 7 runtime and build dependencies from the host system. Key technical instructions include building llama.cpp with AMD GPU targets, configuring GPU layer offloading, and disabling memory mapping (`--no-mmap`) to ensure stability. The roadmap concludes with benchmarking procedures, performance expectations (reaching ~50-52 tokens/second for Llama-2-7B), and troubleshooting tips specific to Ubuntu and AMD's ROCm stack.

A critical distinction emphasized throughout is the difference between traditional dedicated VRAM and APU unified memory. Because the Strix Halo APU shares system RAM, proper allocation of the Graphics Translation Table (GTT) via kernel parameters and modprobe configurations is mandatory to expose the full ~128GB of compute memory to ROCm. The guide recommends against using Ollama due to incomplete AMD support, advocating instead for direct llama.cpp integration within isolated containers.

## Key Takeaways

- BIOS must be configured for 512MB minimum GPU memory, IOMMU disabled, and 85W power mode for optimal performance.
- Ubuntu 24.04 requires kernel 6.16.9+ via the Mainline PPA, along with specific GRUB and modprobe parameters to allocate ~120GB of unified GTT memory.
- Udev rules (`99-amd-kfd.rules`) are critical for granting user access to `/dev/kfd` and `renderD` devices to avoid ROCm out-of-resource errors.
- Podman and Distrobox are used instead of Toolbox to create isolated ROCm 7 RC containers, as Ubuntu's repos lack Toolbox.
- llama.cpp must be built with `-DGGML_HIP_ROCWMMA_FATTN=ON` and run with `--no-mmap` and `-ngl 99` for full GPU offloading.
- For APUs, GTT (Graphics Translation Table) represents the usable unified compute memory (~128GB), not traditional VRAM.
- ROCm with rocWMMA delivers ~2x faster prompt processing than Vulkan at BF16 and excels in long-context scenarios.
